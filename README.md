# CEXpreTrader

Исследовательский стенд (read-only, **paper**, без реальных ордеров) для проверки
торговых стратегий на бессрочных фьючерсах **Gate**. Собирает живую ленту сделок и
стакан по WebSocket, гоняет стратегии на бумаге и честно меряет их нетто после
реалистичных издержек.

## Что внутри

```
Gate watchTrades  (ccxt.pro, WS) → лента сделок ─┬→ ofMomentum (paper-движок)
Gate watchOrderBook (ccxt.pro, WS) → стакан ─────┤    сигнал OFI → вход/выход
                                                 ├→ cex_trades (SQLite, ре-бэктест)
candidateScanner (REST, все перпы Gate, раз в час)┘    отбор по markout/vol
                                                       → of_positions + дашборд :3001
```

- **`src/strategy/ofMomentum.js`** — движок order-flow momentum с реализмом
  исполнения: латентность сигнал→филл, слиппедж входа тейкером, модель очереди на
  мейкер-выходе (цель филлится, только когда через неё прошёл объём ≥ размера),
  онлайн-калибровка порогов OFI/вола, self-отбор символа по трейлинг paper-PnL.
- **`src/strategy/candidateScanner.js`** — REST-скан всех ~820 перпов Gate:
  фильтр по ликвидности/спреду → markout/волатильность → короткий список для
  WS-монитора и `eligible` для торговли.
- **`src/feed/gateFeed.js`** — живой фид Gate по ccxt.pro (`watchTrades` +
  `watchOrderBook`, топ-of-book), динамические подписки: сканер добавляет символы
  на ходу.
- **`src/storage/db.js`** — SQLite (better-sqlite3, WAL, батч-вставки): таблицы
  `cex_trades` (сырьё для офлайн-ре-бэктеста) и `of_positions` (закрытые
  paper-сделки движка).
- **`web/index.html`** — дашборд `:3001`: paper-PnL по символам + статистика
  сбора ленты.

Корневые `_*.mjs` — офлайн-анализаторы (gitignored): ре-бэктест стратегий на
накопленной ленте и исторических свечах.

## Запуск

```bash
cp .env.example .env      # ключи опциональны (см. ниже)
npm install               # better-sqlite3 — нативный, собирается локально
npm start                 # движок + сканер + сбор ленты + дашборд :3001
```

Дашборд: `http://localhost:3001`. Развёртывание на сервере рядом с Gate (AWS
Tokyo) под pm2 — см. [DEPLOY.md](DEPLOY.md).

### Конфигурация
Всё через `.env` (см. `.env.example`), ключевое:
- **Gate API-ключи опциональны** — цены/лента идут по публичному WS; ключи нужны
  лишь для точных taker/funding-комиссий.
- Параметры движка `OFM_*` (окно OFI, пороги, цель/стоп, латентность, размер).
- Параметры сканера `SCAN_*` (мин. объём/спред, глубина, порог markout).

Node ≥ 20.

## Оговорки

- Всё строго **paper**: реальные ордера не выставляются, ключи на вывод не нужны.
- «Сигнал есть» ≠ прибыль. Единственный критерий — нетто после издержек
  (taker/maker + funding + слиппедж + очередь) на walk-forward, а не in-sample.
- Проект рассчитан на Linux-сервер рядом с Gate (Токио): там RTT единицы мс и
  Gate доступен напрямую. Анализ SQLite-файлов сервера делать **на сервере**.
