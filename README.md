# CEXpreTrader

Исследовательский стенд (read-only, **paper**, без реальных ордеров) для поиска
торгуемого edge на бессрочных фьючерсах **Gate**. Собирает живую ленту сделок и
стакан по WebSocket, гоняет стратегии на бумаге и честно меряет их нетто после
реалистичных издержек.

> **Итог исследования (важно):** на доступных рознице данных Gate-перпов
> устойчивого прибыльного edge нашими методами **не найдено**. Реальные сигналы
> существуют, но либо ниже издержек, либо не персистентны на длинной истории.
> Подробнее — раздел [«Результаты»](#результаты). Это отрицательный, но надёжный
> результат: получен через walk-forward + издержки, без подгонки.

## Что внутри

CEX-only. DEX-слой прошлой версии удалён; в бою участвуют:

```
Gate watchTrades  (ccxt.pro, WS) → лента сделок ─┬→ ofMomentum (paper-движок)
Gate watchOrderBook (ccxt.pro, WS) → стакан ─────┤    сигнал OFI → вход/выход
                                                 ├→ cex_trades (SQLite, ре-бэктест)
candidateScanner (REST, все перпы Gate, раз в час)┘    отбор по markout/vol
                                                       → of_positions + дашборд :3001
```

- **`src/strategy/ofMomentum.js`** — движок order-flow momentum с реализмом
  исполнения: латентность сигнал→филл (~20 мс, реальный Токио-RTT), слиппедж
  входа тейкером, модель очереди на мейкер-выходе (цель филлится, только когда
  через неё прошёл объём ≥ размера), онлайн-калибровка порогов OFI/вола,
  self-отбор символа по трейлинг paper-PnL.
- **`src/strategy/candidateScanner.js`** — REST-скан всех ~820 перпов Gate:
  фильтр по ликвидности/спреду → markout/волатильность → короткий список для
  WS-монитора и `eligible` для торговли. DEX-пара не нужна.
- **`src/feed/gateFeed.js`** — живой фид Gate по ccxt.pro (`watchTrades` +
  `watchOrderBook`, топ-of-book), динамические подписки: сканер добавляет символы
  на ходу.
- **`src/storage/db.js`** — SQLite (better-sqlite3, WAL, батч-вставки): таблицы
  `cex_trades` (сырьё для офлайн-ре-бэктеста) и `of_positions` (закрытые
  paper-сделки движка).
- **`web/index.html`** — дашборд `:3001`: paper-PnL по символам + статистика
  сбора ленты.

Корневые `_*.mjs` — офлайн-анализаторы (gitignored): ре-бэктест стратегий на
накопленной ленте и исторических свечах.

## Запуск

```bash
cp .env.example .env      # ключи опциональны (см. ниже)
npm install               # better-sqlite3 — нативный, собирается локально
npm start                 # движок + сканер + сбор ленты + дашборд :3001
```

Дашборд: `http://localhost:3001`. Развёртывание на сервере рядом с Gate (AWS
Tokyo) под pm2 — см. [DEPLOY.md](DEPLOY.md).

### Конфигурация
Всё через `.env` (см. `.env.example`), ключевое:
- **Gate API-ключи опциональны** — цены/лента идут по публичному WS; ключи нужны
  лишь для точных taker/funding-комиссий.
- `EXCLUDE_SYMBOLS=BTC,ETH` — высококонкурентные мейджоры вне универсума.
- Параметры движка `OFM_*` (окно OFI, пороги, цель/стоп, латентность, размер).
- Параметры сканера `SCAN_*` (мин. объём/спред, глубина, порог markout).

Node ≥ 20. Alchemy-ключи в `.env.example` остались от прошлой DEX-версии и
**рабочим CEX-путём не используются**.

## Результаты

Систематически проверены (paper, реалистичные издержки, walk-forward/OOS) и
отвергнуты как суб-издержечные, неторгуемые или неперсистентные:

| Класс стратегии | Итог |
|---|---|
| DEX↔CEX спред / DEX-догоняние | суб-издержки, артефакты (пивот прошлой версии) |
| Маркетмейкинг на CEX | adverse selection, нетто-минус |
| Order-flow momentum | реальный сигнал, но убит очередью+латентностью+слиппеджем |
| Кросс-секц. реверсал / моментум (1ч) | сигнал есть (t~3), но +5 bps < мейкер-издержки |
| Фандинг carry / кросс-биржевой базис-арб | carry-crash / мусор на неликвиде |
| Market-neutral low-vol − BTC (BAB) | отличный на 6-нед OOS, но плоский на 32-мес walk-forward (gross −0.3%) |

**Вывод:** на нашем уровне косто́в и данных Gate-перпы эффективны. Единичный OOS
может «показать» edge (напр. low-vol дал +8% на одном 21-дневном окне) — но
32-месячный walk-forward с издержками гасит его в ноль. Именно поэтому итоговая
валидация всегда walk-forward, а не один сплит.

## Оговорки

- Всё строго **paper**: реальные ордера не выставляются, ключи на вывод не нужны.
- «Сигнал есть» ≠ прибыль. Единственный критерий — нетто после издержек
  (taker/maker + funding + слиппедж + очередь) на walk-forward, а не in-sample.
- Windows-машина: у Node баг на крупных TLS-ответах Gate + IP под баном — код
  предназначен для Linux-сервера рядом с Gate (Токио). Анализ SQLite-файлов
  сервера делать **на сервере** (Windows-сборка better-sqlite3 читает их как
  «malformed»).
